# Notion予定ウィジェットアプリ 仕様まとめ

## 目的

- Notionを開かずに、当日中の「これからの予定」1〜2件をウィジェットに表示
- タップで当日の「今日のスケジュール」ページへ遷移

## 対象 / 前提

- 個人利用
- デバイス：iPhone / iPad / Mac（Apple統一）

## 実装形態

- アプリ本体 + Widget Extension
- ウィジェット：ホーム画面 + ロック画面対応
- サイズ：Small / Medium 両対応

## 認証

- Notion OAuth

## データ源（Notion）

- カレンダーではなく、Notionページ本文から抽出
- 対象ページタイトル：`YYYY-MM-DD 今日のスケジュール`

## ページ内の抽出ルール（MVP）

- 対象範囲：`### Today’s Plan` から `---`（区切り）まで
- 対象行：`- HH:mm-HH:mm タイトル...` の行のみ（時刻が先頭）
- 「これから」の判定：`end > now`（進行中も含む）
- 並び：開始時刻の昇順
- 表示件数：1〜2件

## 進行中予定の扱い

- 進行中（`start <= now < end`）がある場合は、それを1件目として固定表示
- 2件目は「次に来る予定」

## ページが見つからない / 予定がない場合

- 「予定なし」または「ページ未作成」として表示
- アプリからのページ自動作成はしない（MVP）

## タップ時の挙動

- その日のNotionページを開く
- ブロック位置へのジャンプはしない（MVP）

## 更新（WidgetKitの方針）

- 更新頻度：予定の開始/終了タイミングに合わせて更新（方式1）

## データ取得・キャッシュ方針

- アプリ本体がNotion APIから取得し、共有ストレージ（App Group）に保存
- ウィジェットは共有ストレージを読んで表示
- ウィジェット側で毎回Notion APIを叩かない（安定性・制約回避のため）
